# ObelyZK Verifiable Agent Guardrails

*Branch: development | Last updated: April 2026*

---

## What This Is

A cryptographic guardrail system for AI agents. Every action an agent takes is classified by a trained neural network, and the classification produces a GKR+STARK proof that verifies on Starknet. The proof is unforgeable — you can't fake that the safety check ran.

**The pitch:** "Don't trust your agent's guardrails. Verify them."

---

## Architecture

```
Agent wants to act (starkli invoke, transfer, approve)
    ↓
PreToolUse hook intercepts → calls obelyzk_classify
    ↓
Prove-server runs trained MLP (6,240 M31 weights)
    Input:  48 transaction features → M31 matrix (1×64)
    Model:  Linear(64→64) → ReLU → Linear(64→32) → ReLU → Linear(32→3)
    Output: [safe, suspicious, malicious] scores
    ↓
GKR+STARK proof generated (~282ms on A10G)
    ↓
Decision applied:
    score < 40,000  → APPROVE  (execute immediately)
    40,001 - 70,000 → ESCALATE (require human approval)
    > 70,000        → BLOCK    (reject, log proof hash)
    ↓
On-chain firewall updates agent trust score + strike count
    ↓
Audit trail: JSONL at ~/.obelyzk/audit.jsonl
```

## Components Built

### 1. Classifier (Rust — `engine/src/classifier/`)

| File | Purpose |
|------|---------|
| `model.rs` | Graph builder + `build_trained_classifier()` + `load_weights_from_arrays()` |
| `trained_weights.rs` | 6,240 M31 constants (auto-generated by training pipeline) |
| `encoder.rs` | 48-feature transaction encoder (matches Python `features.py` exactly) |
| `prove.rs` | `evaluate_transaction()` — runs MLP + generates GKR proof |
| `types.rs` | `TransactionFeatures`, `ClassifierResult`, `Decision`, thresholds |
| `weight_registry.rs` | Versioned weight management with Poseidon commitment verification |

**48 Input Features (strict order):**

| Index | Feature | Source |
|-------|---------|--------|
| 0-7 | Target address | 8 × 31-bit chunks |
| 8-15 | Value u256 | 8 × 31-bit chunks (low + high) |
| 16 | Function selector | Masked to M31 |
| 17-24 | Calldata prefix | 8 × u32 words |
| 25 | Calldata length | Clamped |
| 26 | Agent trust score | 0-100,000 |
| 27 | Agent strikes | Count |
| 28 | Agent age (blocks) | Since registration |
| 29-32 | Target flags | verified, proxy, source, interaction_count |
| 33-36 | Value features | log2, balance_ratio, max_approval, zero_value |
| 37-40 | Selector features | is_transfer, is_approve, is_swap, is_unknown |
| 41-44 | Behavioral | tx_frequency, unique_targets_24h, avg/max_value_24h |
| 45-63 | Reserved + padding | Zero (MLE alignment) |

### 2. Training Pipeline (Python — `engine/training/`)

| File | Purpose |
|------|---------|
| `train.py` | PyTorch training: class-weighted CE loss, cosine LR, stratified split, M31 export |
| `model.py` | Architecture definition (must match Rust exactly), M31 quantizer |
| `features.py` | 48-feature encoder (must match `encoder.rs` exactly) |
| `data_sources.py` | 10 real exploit patterns + 8 safe DeFi patterns, 75K samples |
| `generate_dataset.py` | v2 synthetic generator (12 patterns, noise, evasion, borderline cases) |
| `adversarial.py` | FGSM robustness, 5 evasion strategies, decision boundary probing |
| `upload_hf.py` | HuggingFace Hub upload with model card + SafeTensors |
| `ciro/api_contract.py` | Type-safe API contract for CIRO data lake integration |
| `ciro/client.py` | Fetch labeled transactions from CIRO → training dataset |

**Training Results (synthetic data):**
- Dataset: 75K samples (40K safe, 20K suspicious, 15K malicious)
- F1 = 0.999 on test set (synthetic data separates cleanly)
- Adversarial: catches 4/4 evasion strategies
- FGSM: fragile at epsilon > 0.05 (expected for 6K-param MLP)

**Current weights are synthetic.** Real weights require running CIRO ingestion (Forta/Etherscan/Starknet/OFAC) then `python -m ciro.client` to fetch labeled data and retrain.

### 3. On-Chain Contracts (Cairo — `elo-cairo-verifier/`)

**Recursive Verifier (`recursive_verifier.cairo`):**
- `register_model_recursive(model_id, circuit_hash, weight_super_root, policy_commitment)`
- `verify_recursive(model_id, io_commitment, stark_proof_data)` → full OODS+Merkle+FRI+PoW
- `get_model_policy(model_id)` → query registered policy commitment
- `get_recursive_verification_count(model_id)`
- Upgrade timelock: propose → 5 min delay → execute
- Real circuit_hash + weight_super_root binding (not placeholders)

**Firewall (`firewall.cairo`):**
- `register_agent(agent_id)` → trust_score=0, strikes=0
- `submit_action(agent_id, target, value, selector, io_commitment)` → action_id
- `resolve_action_with_proof(action_id, proof_hash, io_len, packed_io)` → on-chain verification
- `approve_escalated(action_id)` / `reject_escalated(action_id)` → human-in-the-loop
- Asymmetric EMA trust scoring: alpha=0.5 up (bad), 0.1 down (good)
- Auto-freeze at 5 strikes
- 10 concurrent pending action limit per agent
- Action expiry: 1 hour

**Policy System (`engine/src/policy.rs`):**
- `PolicyConfig` struct with 15 fields
- 3 presets: `strict` (all gates closed), `standard` (streaming-friendly), `relaxed` (dev)
- Poseidon commitment mixed into Fiat-Shamir channel
- Commitment stored on-chain per model via `register_model_recursive`

## How to Pick Up

```bash
# 1. Retrain with real data (when CIRO is ready)
cd engine/training
python -m ciro.client --url $CIRO_URL --api-key $CIRO_API_KEY --output dataset_real.npz
python train.py --dataset dataset_real.npz --epochs 200 --output-dir output
cp output/trained_weights.rs ../src/classifier/trained_weights.rs

# 2. Build and test
cd engine
cargo check --lib --features std
cargo test --lib --features std classifier

# 3. Run prove-server
make build-gpu  # or make build for CPU
make serve

# 4. Test classification
make classify              # safe transaction
make classify-malicious    # malicious transaction

# 5. SDK
cd ../sdk/typescript
npm run build
npx vitest run src/__tests__/firewall.test.ts src/__tests__/mcp-policy.test.ts

# 6. Publish
npm publish                # SDK to npm
cd ../engine/training && python upload_hf.py --repo obelyzk/transaction-classifier-v1  # weights to HF

# 7. Deploy to A10G
make deploy GPU_HOST=your-gpu-host
```
